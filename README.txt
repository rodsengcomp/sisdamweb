============================> Início do Desenvolvimento do Sistema SisdamWeb Access <===================================

Inicio o desenvolvimento do Sisdamweb em meio a uma grande epidemia de Dengue na cidade de São Paulo, na vigilância em
saúde da Suvis Jaçanã-Tremembé recebemos notificações através de fichas em papel e as registramos em sistemas como o
SinanNet e o DengueOnLine, os exames estão sendo coletados atualmente pelo Centro de Controle de Zoonoses (CCZ) e
os resultados são disponibilizados através de planilhas em Excel que extraimos de um site oficial da Coordenação de
Vigilância em Saúde (COVISA), além de digitar as fichas nos sistemas acima mencionados, também utilizamos uma planilha
nomeada de "Controle de Casos" + o ano atual e digitamos manualmente os casos na planilha, a setorização é feita a mão e
os resultados de exames são localizados na planilha de exames e redigitados na planilha de controle de casos.
Esse fluxo repetitivo e pouco eficiente acaba desgastando nosso trabalho e muito além disso toma a maior parte do tempo
disponível, com tamanho problema e desafio de otimizar o trabalho, fiz uma inscrição em um curso de "Access" realizado
pela "Prodam" empresa que presta serviço a Prefeitura de São Paulo e com o conhecimento básico inicio o desenvolvimento
de um sistema capaz de cruzar dados de exame e endereços com a expectativa de otimizar o trabalho, utilizando a
plataforma "Access" da Microsoft utilizando programação em Visual Basic(VBA) estou iniciando o sistema, espero em Deus
que tudo corra bem e que melhore e evolua ao longo dos anos que virão.

------------- Plugins Dependentes -----------

No Eclipse 3.3 / 3.4:

Copie os JARs do plugin de tools / swtbot / 3.3 no diretório do seu Eclipse / dropins (no 3.4)
ou diretório / plugins (no 3.3). Você precisará reiniciar o Eclipse para detectar os novos plug-ins.

No Eclipse 3.5 / 3.6 / 3.7:

Copie os JARs do plug-in de tools / swtbot / 3.5 no diretório / dropins do Eclipse.

Você precisará reiniciar o Eclipse para detectar os novos plug-ins.

O plug-in com.google.gdt.eclipse.maven requer que o M2Eclipse (suporte do Maven) seja instalado. Você pode
feche o projeto ou siga as instruções aqui:
https://sites.google.com/a/google.com/google-plugin-for-eclipse/working-with-maven

O projeto com.google.gdt.eclipse.gph.hge requer que o MercurialEclipse seja instalado. Você também pode
feche o projeto ou instale o MercurialEclipse: http://cbes.javaforge.com/update.

O projeto com.google.gdt.eclipse.gph.subclipse requer que o Subclipse seja instalado. Você também pode
feche o projeto ou instale o Subclipse: http://subclipse.tigris.org/update_1.6.x.

O projeto com.google.gdt.eclipse.gph.subversive requer que o Subversive seja instalado. Você também pode
feche o projeto ou instale o Subversive: http://download.eclipse.org/releases/helios (em Collaboration).

------------ Ordem de classificação do membro ------------

Janela-> Preferências-> Java-> Aparência-> Membros Ordem de Classificação
Não há importação aqui, portanto, faça suas configurações corresponderem:
  configurações / code-style / gwt-sort-order.png

Primeiro, os membros devem ser classificados por categoria.
1) Tipos
2) campos estáticos

3) Inicializadores estáticos
4) Métodos estáticos
5) Campos
6) Inicializadores
7) construtores
8) Métodos

Em segundo lugar, os membros da mesma categoria devem ser classificados por visibilidade.
1) Público
2) Protegido
3) padrão
4) Particular

Terceiro, dentro de uma combinação categoria / visibilidade, os membros devem ser classificados
alfabeticamente.

== Checkstyle ==
O Checkstyle é usado para impor um bom estilo de programação.

1. Instale o plugin Checkstyle v4.x:
   Faça o download de http://sourceforge.net/projects/eclipse-cs/files/Eclipse%20Checkstyle%20Plug-in/v4.4.2/com.atlassw.tools.eclipse.checkstyle_4.4.2-bin.zip/download
   Copie plugins / com.atlassw.tools.eclipse.checkstyle_4.4.2 da extração
   do arquivo zip baixado acima no diretório dropins do seu eclipse
   instalação.

2. Ativar verificações Customizadas do GWT Checkstyle:

Copie "settings / code-style / gwt-customchecks.jar" para:
  <eclipse> /plugins/com.atlassw.tools.eclipse.checkstyle_x.x.x/extension-libraries

3. Importar cheques do GWT:

Janela-> Preferências-> Checkstyle-> Novo ...
Defina o tipo como "Arquivo de configuração externa"
Definir o nome para "cheques GWT" (importante)
Defina a localização para "settings / code-style / gwt-checkstyle.xml".
Sugerido: Marque "Protect Checkstyle configuration file".
Clique OK".

4. Importar verificações do GWT para testes

Repita o passo 3, exceto:
Definir o nome para "Verificações GWT para testes" (importante)
Defina a localização para "configurações / estilo de código / gwt-checkstyle-tests.xml".

== Importando os projetos do plug-in do Google ==

Tendo configurado seu espaço de trabalho apropriadamente, agora você pode importar o
projetos.

Arquivo -> Importar -> Geral -> Projetos Existentes no Espaço de Trabalho

Selecione seu checkout do tronco do Google Eclipse Plugin para ver todos os
Projetos do Eclipse para você importar. Você só deve importar os projetos que
correspondem à versão do Eclipse que você está usando para desenvolvimento e
a plataforma em que você está correndo. Por exemplo, se você tiver o Eclipse 3.4, não
importe um projeto que tenha "3.3" em seu nome. Como outro exemplo, se você é
rodando no Windows, não importe projetos que tenham o readme "win32" ou "macosx"
em seu nome.

== Iniciando o Plugin ==

Uma vez que seus projetos foram importados, vá para o Package Explorer e
clique com o botão direito do mouse no projeto "com.google.gdt.eclipse.suite". Vamos para
"Executar como"> "Aplicativo Eclipse". Outra instância do Eclipse deve ser lançada
executando o GPE!

Nota: A configuração dessas duas variáveis ​​de ambiente fará com que o GWT e o App Engine
SDKs a serem registrados como pacotes do SDK do GPE. No entanto, no modo de desenvolvimento,
só acontece quando os metadados do ambiente de trabalho são criados pela primeira vez. Para ter a bancada de trabalho
pegar as alterações para essas variáveis, vá para a guia Principal no seu lançamento
configuração e marque 'Limpar' em Dados do espaço de trabalho. Note que isso também
remova todos os projetos que você criou no ambiente de trabalho de tempo de execução.
